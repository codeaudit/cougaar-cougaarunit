Adding custom domains to a Cougaar Unit Test Case

The org.cougaar.cougaarunit.PluginTestCase contains the getDomain method. By default this method returns null. In order for a PluginTestCase to utilize a custom domain, the getDomain method needs to be extended to return a list of Component value objects describing the Domain. Here is an example of FooPluginTestCase including the FooDomain withing a run: 

import java.util.ArrayList;
import org.cougaar.cougaarunit.PluginTestCase;
import org.cougaar.cougaarunit.vo.ComponentImpl;
public class FooPluginTestCase extends PluginTestCase{

	public Collection getDomains(){
		ArrayList list = new ArrayList();
		ComponentDescriptionImpl component = new ComponentDescriptionImpl();
		component.setClassName("foo.FooDomain");
		component.setName("FooDomain");
		list.add(new ComponentDescriptionImpl());
		return list;
	}
}
